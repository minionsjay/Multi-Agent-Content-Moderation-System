# 09 Image URL SHA256 Exact Cache

## 测试什么

图片热路径 I3 层：对图片 URL 或 Base64 前缀做 SHA256 哈希，精确匹配缓存。同一个 CDN 链接再次出现 → 直接返回上次的审核结果。

## 测试结果

### Test 1: URL 哈希速度

| URL 长度 | 单次延迟 |
|------|------|
| 38 字符 | 0.4μs |
| 102 字符 | 0.5μs |
| 420 字符 | 0.6μs |

**结论**: 微秒级，可忽略。

### Test 2: 缓存命中率模拟

6,500 次请求（70% 唯一 + 30% 重复 URL）：

```
Hits:   6,280 (96.6%)
Misses: 220   (3.4%)
```

重复 URL 的请求全部命中缓存。唯一图片的首次请求 miss 后写入缓存，后续请求命中。

### Test 3: 哈希碰撞

10,000 个不同 URL → **0 次碰撞**

SHA256 截断到 64-bit（16 字符 hex）在 10K 量级碰撞概率为 0。对于图片 URL 精确去重来说足够安全。
