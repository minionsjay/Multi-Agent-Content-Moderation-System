# 04 EasyOCR Text Extraction

## 测试什么

Image Agent Step 3: 从图片中提取嵌入文字。使用 EasyOCR（ch_sim + en）。

## 测试结果

### Test 1: 冷启动

```
首次提取: 4294ms (包含模型加载)
模型下载: ~450MB (ch_sim 检测器 + 识别器 + en 模型)
```

### Test 2: 热推理延迟

| 文本长度 | 延迟 | 准确率 | 置信度 |
|------|------|------|------|
| 短 (3 chars) "违规" | 308ms | 100% | 0.98 |
| 中 (10 chars) "这是一个测试文本" | 334ms | 100% | 0.89 |
| 长 (30 chars) | 349ms | **61%** | 0.81 |
| English only | 298ms | 100% | 0.81 |
| 中英混合 "加微信WeChat看更多内容" | 342ms | **57%** | 0.69 |

### Test 3: 无文本图片

```
无文本图案: '' → 正确返回空
0 个文本块 ✓
```

### Test 4: 鲁棒性

| 操作 | 延迟 | 提取结果 | 匹配率 |
|------|------|------|------|
| 原始 "违规内容检测" | 367ms | 违规内容检测 | 100% |
| **高斯模糊 2px** | 392ms | **乱码** | **0%** |
| **高斯模糊 4px** | 304ms | **空** | **0%** |
| 低对比度 (×0.5) | 328ms | 违规内容检测 | 100% |

## 关键发现

**EasyOCR 对模糊极其敏感**。2px 高斯模糊就导致完全无法识别。这意味着：
- 低分辨率图片中的文字可能无法被 OCR 提取
- 被打码/马赛克的文字无法识别（符合预期）
- 生产环境需要对图片做预处理（去噪、锐化）

**中英混合识别差**：中英混排文本的识别率只有 57%，英文字母被误认为数字（WeChat → "00000"）。生产需要更好的 OCR 引擎或对 OCR 结果做后处理。

**长文本准确率下降**：30 字文本提取结果只有 61% 匹配。超过 20 字的文本建议分段提取。

## 冷路径中的角色

```
Image Agent:
  1. 下载/解码图片 → ~100ms
  2. NSFW ViT 分类 → ~100ms (POC: 跳过)
  3. EasyOCR 提取 → ~300ms  ← 这里
  4. OCR 文本追加到 state.text → 送入 Text Agent 审核
```

OCR 结果写入 `state.text` 的格式：`原文本 [OCR: 提取的文字]`。Text Agent 后续会对完整文本（包括 OCR 部分）做三轮审核。
